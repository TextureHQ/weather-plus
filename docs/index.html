Weather Plus

Typed weather data from multiple providers, one API.

Weather Plus unifies NWS, OpenWeather, Tomorrow.io, and Weatherbit into a single contract—reshaping each payload, condition code, and error into a shared schema—so your application stays consistent even as providers change.

Why Weather Plus?

Normalized schema & types

Every provider is transformed into the same, strictly typed shape: temperature, humidity, cloud cover, and standardized conditions. Swap providers or add fallbacks without touching downstream code.

Graceful fallback

Configure an ordered provider list. Weather Plus short-circuits on the first success, recording failure taxonomy and latency for downstream observability.

High-hit caching

In-memory or Redis caching with geohash keys maps nearby lookups to the same cell. You get compact storage and strong cache hit rates for geo-centric workloads.

Observability hooks

Outcome reporters log successes, failures, retry hints, and latency so you can emit metrics or alerts without wiring each provider by hand.

Under the hood

  • Provider payloads are reshaped into a single contract and condition taxonomy so your domain logic never sees provider-specific quirks.
  • Errors are normalized (`Unavailable`, `Upstream`, `Validation`, etc.) so fallbacks and retries are deterministic and well-instrumented.
  • Outcome reporters capture latency distribution, success/failure counts, and raw causes for observability and alerting.
  • Fallbacks obey good-citizen behaviour—providers aren’t hammered when unhealthy, and you can layer in your own policies easily.

Quick start

yarn add weather-plus
import WeatherPlus from 'weather-plus';

const weather = new WeatherPlus({
  providers: ['weatherbit', 'openweather', 'nws'],
  apiKeys: {
    weatherbit: process.env.WEATHERBIT_API_KEY!,
    openweather: process.env.OPENWEATHER_API_KEY!,
  },
});

const report = await weather.getWeather(40.7128, -74.0060);
console.log(report.conditions.value);

Configuration overview

Provider ordering

Define priorities with `providers: ['weatherbit', 'openweather', 'nws']`. Weather Plus stops on the first successful provider.

API keys

Supply credentials via the `apiKeys` map. Providers without keys (like NWS) can be omitted.

Caching

Pass a Redis client for shared caching or rely on the built-in in-memory store. Configure TTL and geohash precision to suit your traffic shape.

Observability

Inject your own outcome reporter to pipe metrics into whatever stack you use—Datadog, Prometheus, OpenTelemetry, CloudWatch, you name it.

Fallback and error handling

Weather Plus normalizes provider errors into a shared taxonomy. When every provider fails, the most recent error is re-thrown with full context so you can make decisions (retry, circuit open, alert) confidently.
