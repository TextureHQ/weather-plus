Weather Plus

Typed weather data from multiple providers, one API.

Weather Plus unifies NWS, OpenWeather, Tomorrow.io, and Weatherbit behind a single interface—normalizing disparate payloads and condition codes into one schema—so your application stays consistent even when providers change.

Why Weather Plus?

Multi-provider fallback

Supply an ordered list of providers. Weather Plus will try each one in turn, short-circuiting as soon as a provider succeeds.

Unified schema & types

Every provider is transformed into the same, strictly typed shape: temperature, humidity, cloud cover, and standardized conditions. Swap providers or add fallbacks without changing downstream code.

High-hit caching

Use in-memory or Redis caching with geohash keys. Nearby lookups map to the same cell, delivering high hit rates while keeping the cache compact and predictable.

Reliable fallback

Weather Plus records successes, failure reasons, retry hints, and latency. Healthy providers stay primary; unhealthy ones surface with enough context to debug quickly.

Observability hooks

Inject custom outcome reporters to emit metrics or logs whenever a provider call succeeds or fails. Capture latency distribution and failure taxonomy out of the box.

Under the hood

  • Provider responses are normalized into a single contract, and condition codes are mapped onto Weather Plus’ shared taxonomy.
  • Errors flow through a consistent hierarchy (`Unavailable`, `Upstream`, `Validation`, etc.) so fallbacks and retries are deterministic.
  • Outcome reporters encapsulate success/failure telemetry so you can wire in logging, metrics, or alerting without touching provider code.
  • Provider ordering and retries are “good citizen” friendly: Weather Plus respects rate limits and avoids hammering unhealthy providers.

Quick start

yarn add weather-plus
import WeatherPlus from 'weather-plus';

const weather = new WeatherPlus({
  providers: ['weatherbit', 'openweather', 'nws'],
  apiKeys: {
    weatherbit: process.env.WEATHERBIT_API_KEY!,
    openweather: process.env.OPENWEATHER_API_KEY!,
  },
});

const report = await weather.getWeather(40.7128, -74.0060);
console.log(report.conditions.value);

Configuration overview

Provider ordering

`providers` accepts an array like `['weatherbit', 'openweather', 'nws']`. The first provider to resolve wins; failures automatically fall through to the next provider.

API keys

Use the `apiKeys` map to supply credentials keyed by provider name. Providers without keys (e.g. NWS) can be omitted.

Caching

Pass a Redis client to enable cross-instance caching. Otherwise an in-memory store is used. Adjust TTL and geohash precision as needed.

Observability

Inject a custom outcome reporter to log or emit metrics whenever a provider call succeeds or fails. Capture latency distribution and failure taxonomy out of the box.

Fallback & error handling

Weather Plus normalizes provider errors into a shared taxonomy so you can route retries and observability in a single place. When all providers fail, the most recent error is re-thrown with cause information preserved.
